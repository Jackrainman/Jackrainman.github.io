# 第四周到第八周任务规划

本文用于承接现行 [第二周任务](./第二周任务.md) 和 [第三周任务](./第三周任务.md) 之后的安排。  
对照去年周任务后，当前缺的不是“零散知识点”，而是后续几段完整训练链路还没有重新排期。

## 一、第二周、第三周之后还差哪些任务

第二周、第三周已经覆盖的内容：

- 环境整理、Markdown、基础代码规范
- GPIO、矩阵键盘、串口阻塞式/中断式收发
- C 语言编译过程
- 定时器、PWM、舵机控制

当前还需要补齐的内容：

1. 基础小系统收束  
   把按键、串口、呼吸灯、模式切换、状态输出整理成一个最小分层工程，而不是继续堆例程。
2. 模板工程与 VSCode 开发流程  
   去年这部分放得偏后，今年应提前，否则后面做综合任务时工程会越来越乱。
3. I2C / SPI / OLED / Flash 综合外设链路  
   这是从“会单点功能”走向“会整合系统”的关键一段。
4. ADC 遥控器输入 + 无线链路  
   去年分在第八周，今年可以提前到综合阶段完成，蓝牙和 NRF 二选一即可，不要求双线并行。
5. CAN / 电机 / PID  
   这是项目能力的核心入口，不能再拖到太后面。
6. FreeRTOS / 底盘解算 / 联调  
   需要把通信、控制、执行、反馈真正串起来，形成可验收的机器人子系统能力。

## 二、压缩原则

相比去年，建议按以下方式提速：

- 去年的第四周和第八周前半部分合并，统一作为“系统整合阶段”
- 去年的第五周和第十一至十二周合并，统一作为“项目能力阶段”
- LCD 不再单独设为硬门槛，保留 OLED 即可
- DAC 下沉到提升线，不占主线时间
- 蓝牙和 NRF 二选一，避免在无线模块上平行耗时
- FreeRTOS 不再等到很后面才碰，在做底盘联调前提前接入

## 三、建议周任务分配

### 第四周：基础小系统收束 + 模板迁移

目标：结束“跟着例程做”的状态，开始形成工程意识。

本周任务：

1. 使用 VSCode 或统一模板工程创建新项目，并完成编译、烧录、调试
2. 将第三周的串口中断呼吸灯和 PWM 舵机任务迁移到新工程
3. 增加按键模式切换，至少做出 `IDLE / RUN / CONFIG` 三态
4. 完成最小分层，建议拆出 `bsp_*`、`app_state`、`app_control`
5. 输出一页模块关系图，说明输入、处理、输出链路

验收线：

- 工程能脱离 Cube 自动生成目录独立维护
- 不再把所有逻辑堆在 `main.c`
- 能现场解释新增一个模式时要改哪些文件

参考文档：

- [总则](./总则.md)
- [单片机/03_USART](../单片机/03_USART.md)

### 第五周：I2C / SPI / OLED / Flash

目标：打通“输入数据 -> 存储 -> 显示”的最小综合链路。

本周任务：

1. 跑通 I2C 和 OLED，至少能稳定显示状态信息
2. 跑通 SPI 和 NOR Flash，理解擦除、写入、读取的基本流程
3. 将串口输入的数据写入 Flash，并给每条数据分配编号
4. 在 OLED 上显示当前模式、数据条数、当前写入状态
5. 整理一份笔记，写清楚 I2C、SPI、Flash 地址管理的关键点

验收线：

- 能解释一条串口数据从接收到显示/存储的完整路径
- 能解释为什么 Flash 不能像普通数组一样随便覆盖写
- OLED 刷新逻辑稳定，不出现明显闪屏和卡死

参考文档：

- [寒假任务相关知识/05_I2C](../寒假任务相关知识/05_I2C.md)
- [寒假任务相关知识/06_SPI](../寒假任务相关知识/06_SPI.md)
- [寒假任务相关知识/07_NOR FLASH](../寒假任务相关知识/07_NOR%20FLASH.md)
- [单片机/学习文档](../单片机/学习文档.md)

### 第六周：综合小系统 + ADC 遥控器输入

目标：把前面分散的外设整合成一个可切换模式的小系统，并加入模拟量输入。

本周任务：

1. 使用 ADC 读取双摇杆或等效模拟输入
2. 在 OLED 上实时显示 X/Y 数值与按键状态
3. 将串口输入、按键输入、ADC 输入统一整理为事件入口
4. 完成综合小系统：
   - 串口输入
   - 按键切换模式
   - Flash 存储
   - OLED 显示
   - ADC 输入显示
5. 提升线可增加蓝牙或 NRF 二选一发送链路

验收线：

- 状态切换清晰，不靠大段 `if-else` 硬拼
- 能说明输入层、业务层、显示层各自职责
- 至少完成一类无线模块的预研，或给出接口预留方案

参考文档：

- [寒假任务相关知识/04_DMA](../寒假任务相关知识/04_DMA.md)
- [单片机/学习文档](../单片机/学习文档.md)

### 第七周：CAN / 电机 / PID 入门

目标：进入项目能力主线，建立通信到执行器的闭环意识。

本周任务：

1. 学习 CAN 协议，完成基础收发测试
2. 阅读并使用战队现有电机驱动和 `can_list` 相关代码
3. 跑通至少 1 路电机的启停、目标值下发与反馈解析
4. 学习 PID 基本原理，完成速度环或位置环中的一种
5. 输出通信映射表和控制周期说明

验收线：

- CAN 初始化、发包、收包链路清楚
- 能说明电机不动时先查哪些点
- 能解释 PID 三项各自的作用和当前参数调整思路

参考文档：

- [CAN/CAN总线基础](../CAN/CAN总线基础.md)
- [CAN/CAN_List](../CAN/CAN_List.md)
- [CAN/CAN_List代码](../CAN/CAN_List代码.md)
- [Motor/电机驱动代码详细讲解](../Motor/电机驱动代码详细讲解.md)
- [ControlTheory/03-PID控制算法原理与应用](../ControlTheory/03-PID控制算法原理与应用.md)

### 第八周：FreeRTOS + 底盘解算 + 遥控联调

目标：形成一个真正可演示的机器人子系统闭环。

本周任务：

1. 学习 FreeRTOS 的任务、阻塞、调度基础
2. 把输入、控制、输出按任务或模块拆开
3. 学习底盘解算，至少理解全向轮或麦轮中的一种
4. 使用遥控器输入控制底盘速度指令
5. 完成“遥控器输入 -> 解算 -> 电机控制 -> 反馈”的联调演示

验收线：

- 至少能稳定控制一个底盘模型完成前进、横移、旋转中的基本动作
- 能解释任务划分依据和控制周期设置理由
- 能给出一份联调问题清单与修复记录

参考文档：

- [FreeRTOS/01_调度](../FreeRTOS/01_调度.md)
- [Chassis/底盘解算](../Chassis/底盘解算.md)

## 四、执行建议

为避免节奏又慢回去年，建议统一执行以下规则：

1. 每周只保留一个主作业，不再同周塞太多平行知识点
2. 每周验收都要求提交工程、笔记、AI 使用记录、现场演示
3. 通关线未过，不进入下一周主线，只允许补当前周
4. 快的同学进入提升线：
   - 第五周提前做 DMA 接收
   - 第六周提前做无线发送
   - 第七周提前做双电机
   - 第八周提前做双任务或多任务联调

## 五、最终对应关系

按这个排期，去年分散在第四、第五、第八、第十一至十二周的内容，会被压缩为现在第三周之后的五周主线：

- 第四周：工程化收束
- 第五周：综合外设链路
- 第六周：综合小系统
- 第七周：CAN 与电机控制
- 第八周：FreeRTOS 与底盘联调

如果执行顺利，整体进度会明显快于去年，但仍保留每周都可现场验收的强约束。
