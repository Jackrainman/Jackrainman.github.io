# Robocon 电控培养总则

## 1. 适用范围与目标

本总则适用于电控新人培养的全部阶段，目标是让成员从“能完成基础程序”逐步成长为“能参与机器人整机开发”。

阶段终态：

- 预备阶段：环境、工具链、代码规范和最小外设基础清理完毕
- 第一阶段：会独立完成基础控制链并解释程序结构
- 第二阶段：会组织系统工程并完成多模块整合
- 第三阶段：会在真实机器人场景完成通信、控制、执行链路联调

## 2. 培养结构与推进机制

培养采用 `预备阶段 + 三阶段主线`：

1. 预备阶段：统一环境、软件、规范与基础外设
2. 第一阶段：基础能力阶段
3. 第二阶段：系统整合阶段
4. 第三阶段：项目能力阶段

推进规则：

- 任务可以按周发布，但晋级按阶段通关
- 预备阶段未通过，不进入第一阶段
- 每个阶段统一分为：`通关线 / 提升线 / 预研线`
- 通关线未通过，不进入下一阶段
- 提前完成者可进入预研线，但仍需补齐本阶段提交物
- 未过线成员进入补救路径，不以自然周数直接淘汰

## 3. 统一提交规范

每次验收前至少提交：

1. 工程文件（可编译、可复现）
2. 周笔记（`WeekX_笔记.md`）
3. AI 使用记录
4. 现场验收记录

笔记最低要求：

- 本周学习内容
- 程序结构图或流程图
- 至少 2 个问题与解决过程
- 仍未解决的问题

AI 使用记录最低要求：

- 提问内容
- 采用了哪些建议
- 自己做了哪些改动
- 仍无法解释的点

## 4. 统一验收与评分

验收采用面对面方式，固定检查 4 项：

A. 功能是否可以现场稳定复现  
B. 目录是否清晰、是否有分文件意识  
C. 是否能解释核心逻辑  
D. 是否能完成临时小改需求

建议评分：

| 分值 | 定义 |
| --- | --- |
| 0.0 | 未完成或不可运行 |
| 0.6 | 现象部分完成，但讲不清逻辑 |
| 0.8 | 基本完成，能解释部分逻辑，但改动能力偏弱 |
| 1.0 | 通关线完成且能解释核心逻辑 |
| 1.2 | 完成稳定，结构清晰，能扩展或完成临时改动 |

统一口径：

- 现象完成不等于通过
- 能解释、能改动才算真正通过

## 5. AI 使用边界

允许：

- 查概念、报错、外设原理、调试方向
- 辅助梳理函数与模块关系
- 协助总结问题和整理笔记

不允许：

- 直接生成整套作业并提交
- 不理解即照抄
- 无法解释初始化、中断、主循环职责

## 6. 学长验收 SOP（3 到 5 分钟）

1. 看 20 秒目录：是否分文件、是否全塞 `main.c`
2. 现场跑 60 秒功能：看现象是否稳定、链路是否完整
3. 抽问 2 到 3 个问题：从当周题库抽查
4. 临时改 1 个小点：看是否能完成最小修改
5. 给结论：分值 + 1 句关键反馈
