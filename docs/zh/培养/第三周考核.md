# 第三周考核

这份文档基于 [第三周任务](./第三周任务.md) 和 [第三周任务学习笔记](./第三周任务学习笔记.md) 整理，目的是给提问人一份顺手的“问题 + 参考答案”清单。

默认原则：

1. 优先考“能不能讲清楚主线”，不是只背名词。
2. 参考答案是答题方向，不要求逐字复述。
3. 可以先问基础题，再按回答质量往下追问。

---

## 一、必须问：主线是否清楚

### 1. C 源文件从 `.c` 到最终二进制，大致经历哪几步？

参考答案：

通常是预处理、编译、汇编、链接；嵌入式里常常还会把 `ELF` 再转成 `BIN/HEX` 用于烧录。

### 2. 预处理主要做什么？

参考答案：

主要做 `#include` 展开、`#define` 宏替换、条件编译处理、删除注释。它本质上是在把源代码整理成更完整的 C 文本，还没有真正生成目标平台指令。

### 3. 为什么 `.h` 文件一般不单独编译？

参考答案：

因为 `.h` 更多是声明和宏定义，通常是通过 `#include` 并入 `.c` 文件后，随着 `.c` 一起参与编译。

### 4. 编译阶段和汇编阶段分别做什么？

参考答案：

编译阶段把 C 代码变成目标平台对应的汇编代码；汇编阶段再把汇编指令翻译成机器码，生成目标文件 `.o`。

### 5. 为什么一个工程里会有很多 `.o` 文件？

参考答案：

因为通常每个 `.c` 文件会先独立编译成一个 `.o`，最后由链接器把多个 `.o` 和库文件合并起来。

### 6. `.o` 已经是二进制了，为什么还不能直接烧录？

参考答案：

因为它往往只是一个二进制片段，很多符号地址还没最终确定，多个文件也还没合并，内存布局也没完全安排好。

### 7. 链接器到底解决了什么问题？

参考答案：

链接器主要做三件事：合并代码段和数据段、解析符号引用、完成重定位，最终生成可以运行或可进一步烧录的映像。

### 8. 什么叫交叉编译？

参考答案：

就是在一个平台上编译，生成给另一个平台运行的程序。比如在电脑上用 `arm-none-eabi-gcc` 编译，生成给 STM32 跑的程序。

### 9. PC 编译器和嵌入式编译器最大的区别是什么？

参考答案：

最大的区别是目标平台和运行环境不同。PC 编译器通常面向有操作系统的 `x86/x64` 平台；嵌入式编译器通常面向 MCU，常常没有完整操作系统，需要自己关心启动文件、内存布局和中断向量表。

### 10. 为什么嵌入式项目更依赖链接脚本？

参考答案：

因为必须明确 `.text`、`.data`、`.bss`、中断向量表分别放到哪块 `FLASH/RAM`，否则程序可能根本跑不起来。

### 11. 串口三种常见中断通常指哪三个？

参考答案：

入门阶段常说的是 `RXNE`、`TXE`、`TC`。有些资料也会把 `IDLE` 作为重点补充。

### 12. `RXNE` 表示什么？

参考答案：

表示接收寄存器非空，也就是收到了一个新字节，CPU 可以来读数据了。

### 13. `TXE` 和 `TC` 的区别是什么？

参考答案：

`TXE` 表示发送数据寄存器空了，可以继续装下一个字节；`TC` 表示最后一个字节连同最后一位也真正发完了。`TXE` 不等于整帧发完。

### 14. 为什么串口接收中断里要及时把数据读出来？

参考答案：

因为数据已经进了接收寄存器，中断里要把它取走并转存到软件缓冲区，否则可能影响后续接收，严重时会丢数据或引发错误标志处理问题。

### 15. DMA 的本质是什么？

参考答案：

DMA 是直接存储器访问，本质上是让硬件帮忙在外设和内存之间搬数据，减少 CPU 逐字节参与。

### 16. DMA 为什么比“每字节一个中断”更高效？

参考答案：

更高效的根本原因不是单次搬运魔法般更快，而是减少了 CPU 频繁进中断和逐字节搬运的开销。

### 17. 串口接收常见三种方式是什么？

参考答案：

轮询、中断、DMA。轮询最简单但最占 CPU；中断更高效；DMA 最适合连续数据流。

### 18. 基本定时器和通用定时器的区别是什么？

参考答案：

基本定时器主要用于纯定时和更新中断，结构简单；通用定时器功能更多，除了定时，还支持输入捕获、输出比较、PWM 等。

### 19. 为什么基本定时器通常不能直接输出 PWM？

参考答案：

因为它没有通用定时器那套完整的输出比较/PWM 通道能力，主要是做时间基准和周期事件。

### 20. 定时器更新频率公式是什么？

参考答案：

常用公式是 `f_update = 定时器时钟 / (PSC + 1) / (ARR + 1)`。

### 21. `PSC` 和 `ARR` 各控制什么？

参考答案：

`PSC` 控制预分频，也就是先把输入时钟降下来；`ARR` 控制计数上限，也就是一个更新周期有多长。

### 22. PWM 里 `ARR` 和 `CCR` 分别决定什么？

参考答案：

`ARR` 主要决定周期，`CCR` 主要决定高电平持续时间，所以 `CCR / (ARR + 1)` 对应占空比。

### 23. 为什么改 `CCR` 就能改占空比？

参考答案：

因为在同一个 PWM 周期内，比较值改变了输出翻转时刻，也就改变了高电平所占比例。

### 24. 串口中断和外部中断的核心区别是什么？

参考答案：

串口中断来自 USART 外设内部事件，比如收到数据、发送完成；外部中断来自 GPIO 引脚电平变化，比如上升沿、下降沿。

### 25. 任务里“串口中断控制呼吸灯频率”的关键考点是什么？

参考答案：

关键不是把灯点亮，而是讲清楚数据接收链路：串口收到命令，中断读出数据，软件解析命令，再修改控制呼吸灯频率的参数。

### 26. 任务里“PWM 控舵机”的关键考点是什么？

参考答案：

关键是能说清楚通用定时器如何输出 PWM、PWM 参数如何对应舵机角度、按键是如何改变目标占空比或脉宽的。

## 二、建议问：理解是否扎实

### 27. 如果只有函数声明没有函数定义，编译和链接分别会怎样？

参考答案：

编译阶段可能能过，因为语法上看到了声明；链接阶段大概率过不了，因为找不到真正定义，符号无法解析。

### 28. 为什么说“预处理后还是 C 文本，但编译后已经和目标架构相关”？

参考答案：

预处理只是文本级展开，不关心具体 CPU；编译后生成的是某个目标架构的汇编或中间结果，已经和 ARM、x86 之类的平台绑定了。

### 29. 为什么同样一份 C 代码，换个 CPU 架构，生成的汇编会不一样？

参考答案：

因为不同架构的指令集、寄存器组织、调用约定都不同，编译器必须按目标平台规则生成代码。

### 30. 为什么嵌入式比 PC 更关心代码体积和 RAM 占用？

参考答案：

因为 MCU 的 `FLASH` 和 `RAM` 资源通常很有限，超了就放不下或运行异常，而 PC 的资源通常宽裕得多。

### 31. 为什么 `TXE` 常用于连续发送，而 `TC` 常用于收发切换？

参考答案：

因为 `TXE` 说明可以继续塞数据，适合流水发送；`TC` 说明线路上最后一位也发完了，适合需要严格确认“真的结束”的场景，比如 `RS485` 方向切换。

### 32. 什么时候会优先选 `DMA + IDLE`，而不是纯 `RXNE`？

参考答案：

在接收不定长、连续、高速的数据流时更适合 `DMA + IDLE`，因为 DMA 负责搬运，`IDLE` 负责判定一帧结束，CPU 负担更小。

### 33. 如果串口数据量很大，只靠 `RXNE` 中断可能有什么问题？

参考答案：

中断会非常频繁，CPU 占用变高，系统中其他任务的实时性会受影响，严重时可能来不及处理而丢数据。

### 34. “DMA 快”是不是表示 DMA 的硬件时钟一定比 CPU 更快？

参考答案：

不一定。更准确地说，DMA 的优势是把搬运这件事从 CPU 手里拿走，让 CPU 不用频繁插手。

### 35. 为什么通用定时器能做 PWM，本质上依赖的是什么能力？

参考答案：

本质上依赖输出比较功能和通道输出能力。定时器在计数过程中和 `CCR` 做比较，按设定规则改变输出电平。

### 36. 如果把定时器周期算错了，PWM 控舵机会出现什么现象？

参考答案：

可能会导致舵机抖动、角度不准、转动异常，甚至因为脉宽和周期不符合舵机要求而无法正常工作。

### 37. 基本定时器“500 ms 翻转一次 LED”的例程，为什么中断里还要清标志？

参考答案：

因为更新事件触发后会置位标志，不清掉可能导致中断反复进入或状态判断异常。

### 38. 串口中断和外部中断都要走 NVIC，它们为什么仍然不是一回事？

参考答案：

因为 NVIC 只是统一管理中断响应的控制器，真正的触发源和寄存器路径仍然分别来自 USART 外设和 EXTI/GPIO 路径。

### 39. 为什么说“中断是通知机制，不是业务本身”？

参考答案：

因为中断的作用是告诉 CPU 某个事件发生了，真正的业务逻辑是你在中断里或主循环里对这个事件做什么处理。

### 40. 如果学弟说“DMA 完全不占用 CPU”，你会怎么纠正？

参考答案：

这句话不严谨。DMA 只是减少 CPU 的搬运参与，但配置 DMA、处理完成事件、解析数据这些事仍然需要 CPU。

### 41. 为什么很多工程会把串口中断里只做“搬运和置标志”，把复杂解析放到主循环？

参考答案：

因为中断里应尽量短小，避免占用太久影响系统实时性。复杂处理放到主循环或任务里更稳妥。

### 42. 舵机控制里“角度”和“占空比”为什么不能简单认为是线性、通用、固定关系？

参考答案：

因为不同舵机的控制周期、最小脉宽、最大脉宽可能不同，实际角度和脉宽对应关系也会有偏差，通常要按具体舵机标定。

### 43. 为什么“能点亮 LED”不等于“定时器理解到位”？

参考答案：

因为点亮 LED 只能说明现象对了，不代表能解释时钟来源、分频、计数周期、中断触发条件这些真正原理。

### 44. 为什么嵌入式里常说“先看数据流，再看控制流”？

参考答案：

因为很多外设问题本质上是“数据从哪里来，经过哪里，到哪里去”，先理清寄存器、缓冲区、DMA、内存之间的数据路径，代码就更容易理解。

## 三、可选刁钻题：结合生活和常识

### 45. 你知道你手机大概率是什么 CPU 架构吗？

参考答案：

大概率是 ARM 架构。因为手机 SoC 长期以 ARM 生态为主，功耗和集成方案很适合移动设备。

### 46. 你的电脑 CPU 和 STM32 的 CPU 架构一样吗？

参考答案：

通常不一样。电脑常见是 `x86_64`，STM32 常见是 ARM Cortex-M。它们都可能属于“CPU”，但面向的场景、指令集和资源条件不同。

### 47. 电脑的 CPU 和 GPU 架构一样吗？为什么？

参考答案：

通常不一样。CPU 更强调低延迟、通用控制和复杂分支处理；GPU 更强调大规模并行吞吐，适合大量相似计算。两者都做计算，但优化目标不同。

### 48. 为什么同样叫“处理器”，手机 SoC、电脑 CPU、单片机 MCU 差异会这么大？

参考答案：

因为它们的设计目标不同。手机强调功耗、集成和多媒体；电脑强调通用性能；MCU 强调实时控制、成本、可预测性和外设集成。

### 49. 为什么你写 PC 程序时几乎不关心代码放在哪块内存，但写 MCU 程序要关心？

参考答案：

因为 PC 有操作系统和虚拟内存做抽象，很多细节被屏蔽了；MCU 资源直接暴露给程序，布局不对就可能无法启动或运行异常。

### 50. 如果把“轮询、中断、DMA”类比成生活场景，可以怎么理解？

参考答案：

轮询像老师每分钟主动去每个同学桌前问一次“你做完没”；中断像同学做完后举手叫老师；DMA 像安排了助教自动收作业，老师只在关键节点看结果。

### 51. 为什么机械按键更像“抖动信号源”，而不是一次干净变化？

参考答案：

因为机械触点在按下和松开瞬间会反复弹跳，电平不是一次性稳定翻转，所以按键处理中常需要消抖。

### 52. 为什么“发消息成功”和“消息已经被对方看见”不是一回事？它像 `TXE` 还是 `TC`？

参考答案：

更像 `TXE` 和 `TC` 的区别。你把消息提交出去，不等于链路上所有步骤都结束；真正“最后一位发完”才更接近 `TC` 的语义。

### 54. 为什么舵机控制更适合用定时器硬件输出 PWM，而不是软件延时翻转 IO？

参考答案：

因为硬件定时器输出更稳定、抖动更小、占用 CPU 更少，软件延时容易受其他任务影响而导致波形不准。

### 55. 为什么“串口”和“外部中断”都可以触发程序反应，但工程里不能混着讲？

参考答案：

因为一个是通信外设事件，一个是引脚边沿事件，硬件路径、配置方式、典型用途都不一样。只看“都能打断 CPU”会把问题说得过于粗糙。

### 56. 如果把一个串口接收程序写成“每来一个字节就做完整协议解析”，你觉得风险是什么？

参考答案：

风险是中断里处理过重，导致实时性下降，甚至影响后续字节接收。更常见的做法是先缓存，再在主循环或任务里解析。

### 57. 为什么很多通信协议都离不开“缓冲区”？

参考答案：

因为数据到达和程序处理往往不同步，缓冲区的作用就是在两者之间做暂存和解耦。

### 58. 手机充电时，系统周期性读电池信息更像轮询还是中断？

参考答案：

如果是定时去读状态，更像轮询；如果是电池管理芯片在异常或状态变化时主动通知，更像中断。

### 59. 为什么说 DMA 不是“更高级的中断”，而是不同层面的机制？

参考答案：

因为 DMA 解决的是“谁来搬数据”的问题，中断解决的是“事件发生后如何通知 CPU”的问题。DMA 完成后照样可能要靠中断通知 CPU。

### 60. 为什么你电脑上能同时开很多程序，而 MCU 程序常常还在精打细算几个中断和一个缓冲区？

参考答案：

因为资源量级完全不同。电脑有更强的 CPU、更大的内存和操作系统调度；MCU 往往资源有限，而且很多任务还要满足实时性和可预测性。

## 四、适合临场连续追问的问题链

### 问题链 A：编译流程

1. `.c` 怎么变成 `.bin`？
2. 哪一步开始和 CPU 架构绑定？
3. `.o` 为什么还不能直接跑？
4. 链接器怎么知道程序该放到哪块内存？
5. 为什么这个问题在 PC 上不太常被你手动关心？

参考答案：

先看总流程，再追到“编译和汇编的区别”，然后追“链接脚本决定布局”，最后落到“PC 有 OS 抽象，MCU 需要自己关心内存布局”。

### 问题链 B：串口接收

1. 串口收到 1 个字节后发生了什么？
2. `RXNE` 为什么会触发？
3. 中断里为什么要读寄存器？
4. 数据放哪？
5. 数据量大了为什么不继续用这个方案？

参考答案：

硬件收字节、置位标志、NVIC 响应、进中断、读寄存器、转存缓冲区。数据量大后中断太频繁，所以会考虑 DMA。

### 问题链 C：PWM 控舵机

1. 为什么选通用定时器而不是基本定时器？
2. PWM 周期谁决定？
3. 占空比谁决定？
4. 改哪个参数会影响角度？
5. 为什么不用软件延时来“手搓 PWM”？

参考答案：

因为需要 PWM 通道能力；`ARR` 定周期，`CCR` 定占空比；实际常通过改脉宽对应参数控制角度；硬件定时器比软件翻转更稳。

## 五、给提问人的使用建议

### 必须问

1. 先问总流程题，看他能不能自己把主线串起来。
2. 再问一个串口题和一个定时器题，看能不能说出事件链路。
3. 最后问一个“为什么”，区分他是背答案还是理解了。

### 建议问

1. 如果对方答得太短，就追问“为什么”与“如果不用这个方案会怎样”。
2. 如果对方只会背定义，就让他描述一个例程从初始化到运行的完整过程。
3. 如果对方只会说现象，就让他解释寄存器、缓冲区、DMA、NVIC 分别起什么作用。

### 可选优化

1. 可以从生活问题切入，再拉回专业概念，这样更容易看出他是否真的理解。
2. 可以故意问一些容易混淆的点，比如 `TXE` 和 `TC`、外部中断和串口中断、轮询和 DMA。
3. 如果要提高区分度，可以要求他用自己的话解释，不允许只报名词。
