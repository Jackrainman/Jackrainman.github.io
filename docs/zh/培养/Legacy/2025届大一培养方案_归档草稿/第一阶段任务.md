# 第一阶段任务（基础能力阶段）

## 1. 阶段定位

本阶段建立在预备阶段通过基础上，目标是把“会跟着例程做”推进到“会独立完成基础控制链并解释程序结构”。

通过后至少应具备：

- 能独立完成串口中断、定时器、PWM 等基础控制链
- 能理解参数与现象之间的对应关系
- 能进行最小分层，避免所有逻辑堆在 `main.c`
- 能完成一个小型整合作业并解释模块关系

推荐节奏：2 到 4 周。任务仍可按周发布，但是否进入下一阶段按通关结果决定。

## 2. 阶段统一规则

- 进入条件：预备阶段通过
- 验收方式：面对面验收，按总则评分
- 本阶段统一结构：`通关线 / 提升线 / 预研线`
- 未过线成员走补救路径，已过线成员可进入预研线

---

## 节点 1：串口中断、定时器、PWM

### 本节点目标

1. 学会串口中断收发基础
2. 学会定时器参数配置（PSC/ARR）
3. 学会 PWM 核心参数与舵机控制
4. 建立“输入 -> 处理 -> 输出”的最小控制链意识

### 通关线

1. 使用串口中断控制呼吸灯频率
2. 使用 PWM 控制舵机角度（按键或命令切换）
3. 输出一页参数说明笔记，说明频率、占空比、现象映射关系

### 提升线

- 串口命令控制舵机
- 频率或角度的上下限保护
- 把控制逻辑集中到 `app_control.c/h` 这类文件

### 预研线

- 非阻塞主循环
- 串口接收缓存（非单字节）
- 舵机平滑控制

### 验收问题（抽 2 到 3 个）

- PSC 和 ARR 改动分别影响什么？
- PWM 为什么可以同时用于呼吸灯和舵机？
- 串口中断和外部中断的核心区别是什么？
- DMA 和中断在这里的主要差异是什么？

### 节点通过标准

- 串口中断与 PWM 功能稳定
- 能解释参数与现象的关系
- 有最小分文件意识

---

## 节点 2：基础小系统收束

### 本节点目标

1. 把前面学到的基础能力整合成一个小系统
2. 强化状态划分、模块边界和职责分离
3. 为第二阶段的模板迁移和整合任务打入口

### 通关线

小系统至少包含：

- 运行心跳（LED 或呼吸灯）
- 串口命令输入（2 到 3 条）
- 按键切换模式（如 `IDLE/RUN/CONFIG`）
- 串口状态输出
- 分文件组织（建议 `bsp_* + app_state`）
- 一张模块图（输入 / 处理 / 输出）

### 提升线

- 运行时间计数并显示或打印
- 软件 IIC 或 SPI 最小跑通
- 对状态切换增加保护条件

### 预研线

- 基于统一模板迁移该小系统
- 提前阅读 OLED、Flash 或状态机资料
- 尝试把命令解析改成事件驱动

### 验收问题（抽 2 到 3 个）

- 你的状态是如何划分的，为什么这样分？
- 新增一个模式，需要改哪些位置？
- 为什么需要 `app_state.c/h` 这类中间层？
- 中断、主循环、PWM 更新职责如何分离？

### 节点通过标准

- 小系统稳定运行
- 状态边界基本清晰
- 代码已具备初步分层

## 3. 阶段通关标准

满足以下全部条件：

1. 基础外设与控制链（串口中断 / 定时器 / PWM）可独立完成并解释
2. 能说明程序结构与模块调用关系
3. 能通过抽查问题并完成临时小改
4. 小整合作业可现场稳定复现

## 4. 未过线补救路径

- 先补节点 1，再补节点 2，不跨节点补
- 下次验收优先看：功能稳定、结构解释、最小改动能力
- 补救目标不是做更多，而是先补齐通关线
