# 培养方案调整说明（仅内部）

## 1. 这次调整解决什么问题

今年的目标不是通过高强度任务筛掉慢同学，而是在不放弃任何新人的前提下，解决以下问题：

- 前两周任务容易被当成整套培养方案，后续分层不清
- 快的同学容易空转，慢的同学也只是延后暴露问题
- 学长带教资源紧张，验收口径容易不统一
- 新人容易同时看到多份文档，不知道哪份是当前执行版本

## 2. 这次调整的核心结论

- 不是完全推翻旧方案
- 保留三阶段主骨架
- 前两周正式定义为“预备阶段”
- 从第 3 周开始，任务仍可按周发布，但培养主线按阶段推进

简化后结构如下：

1. 预备阶段：清环境、清规范、清基础逻辑
2. 第一阶段：基础控制链和小整合
3. 第二阶段：模板、模块化、综合整合
4. 第三阶段：CAN、电机、PID、底盘与联调

## 3. 为什么这样改，而不是完全重构

- 三阶段结构本身没有问题，问题在于阶段边界和入口不清
- 如果现在完全重构，队内阻力会很大，也会浪费已有文档积累
- 当前更合理的方式是：保留主骨架，重构内部节奏、验收和流转规则

本次修改的本质是：

- 外层结构保留
- 入口收束
- 验收方式重写
- 阶段内部任务重新组织

## 4. 当前主文档与归档文档的职责

当前主文档：

- `培养/培养方案.md`：唯一入口
- `培养/总则.md`：统一规则、评分、AI 边界、验收 SOP
- `培养/预备阶段任务.md`：前两周统一起跑线
- `培养/第一阶段任务.md`：基础能力阶段
- `培养/第二阶段任务.md`：系统整合阶段
- `培养/第三阶段任务.md`：项目能力阶段

归档文档：

- `培养/Legacy/2024届大一培养方案_定稿/`：上一届正式方案
- `培养/Legacy/2025届大一培养方案_归档草稿/`：本届后续暂时不用的草稿、替代稿、讨论稿

## 5. 对队友的统一说法

- 不是否定前两周任务，而是给它们一个更准确的位置
- 前两周继续保留，但它们只承担“预备阶段”的职责
- 从第 3 周开始，不能再继续用“统一拖进度”的方式推进
- 周发布可以保留，但阶段通关必须写进文档和验收规则
- 综合大作业不放第一阶段，放第二阶段更合理，因为它考的是整合而不是基础

## 6. 验收口径为什么要改

这次统一改为面对面验收，原因是：

- 视频对当前组织方式没有必要
- 面对面更容易看出是不是自己真正做懂了
- 学长更容易通过目录、抽问和现场小改快速判断真实水平

统一验收口径：

- 看目录
- 跑功能
- 抽问题
- 改一个小点

评分统一保留：

- `0.0`
- `0.6`
- `0.8`
- `1.0`
- `1.2`

其中 `0.8` 很重要，用来承接“基本做出来了，但还没有真正通关”的同学。

## 7. 目前还有哪些可继续优化

- 补一份统一题库，减少不同学长抽问差异
- 补一份高频 bug 清单，方便未过线同学自查
- 补一份现场验收记录模板，方便后续跟踪补救
- 补统一资料包入口，减少新人生找资料
- 等本届培养跑一轮后，再决定是否进一步压缩或拆分第二阶段
