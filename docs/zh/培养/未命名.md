# 预备阶段任务（前两周）

## 1. 阶段定位

本阶段对应当前已确定的前两周任务，用于统一新人起跑线。

目标不是拉开能力差距，而是先把以下底座清理干净：

- Windows 环境与软件安装习惯
- Markdown 笔记与提交习惯
- HAL 工程基本结构理解
- GPIO、串口、外部中断等最小外设能力
- 代码规范和面对面验收习惯

通过后至少应具备：

- 能独立完成环境搭建、编译、下载、串口联通
- 能说清 `main / 初始化 / while(1) / 中断回调` 的基本职责
- 能完成 LED、串口回显、按键中断这类最小任务
- 代码目录和命名不至于完全失控

## 2. 执行说明

- 本阶段内容已确定，不再额外推翻重排
- 当前前两周任务默认归入预备阶段
- 预备阶段通过后，再进入后续三阶段主线
- 验收方式统一按总则执行：面对面、看目录、抽问、现场小改

## 3. 当前任务内容

### 节点 A：环境、工具与基础认知

通关线：

1. 完成正点原子相关例程 13-16 的学习与笔记整理
2. 完成电脑环境自查，处理中文用户名、流氓软件、安装路径混乱等问题
3. 会使用 Markdown 编写笔记
4. 了解战队代码规范，重点关注命名、注释和缩进

提升线：

- 用自己的话画出 HAL 工程结构图
- 能指出用户代码应写在什么位置，哪些代码是自动生成的
- 能说明为什么环境和目录混乱会影响后续开发

### 节点 B：最小功能链路

通关线：

1. 两个 LED 不同步闪烁（如 LED0 闪两下，LED1 闪一下）
2. 串口 1 回显电脑通过串口工具发来的消息
3. 按键中断触发串口输出，说明哪个按键被按下

提升线：

- 把 LED 或串口逻辑做最小分文件
- 解释外部中断与轮询的区别
- 写一页编译流程笔记，说明源文件到可执行文件的大致过程

预研线：

- 尝试从零新建最小 HAL 工程
- 尝试把 LED、按键、串口拆成 `bsp_*` 风格文件
- 提前阅读定时器、PWM、串口中断资料

## 4. 验收问题（抽 2 到 3 个）

- 初始化为什么通常写在 `while(1)` 之前？
- 外部中断和轮询的核心区别是什么？
- 串口参数不一致为什么会乱码？
- 为什么不建议把所有逻辑都堆在 `main.c`？
- 如果让你把 LED 闪烁节奏改掉，你要改哪部分代码？

## 5. 通关标准

满足以下全部条件即通过预备阶段：

1. 环境、驱动、串口工具可正常使用
2. LED、串口回显、按键中断功能可现场稳定复现
3. 能解释 HAL 工程主结构和基本控制流程
4. 面对面抽问与现场小改不过度依赖照抄例程

## 6. 未过线补救路径

- 先补齐环境和工具问题，再补代码问题
- 优先补：LED、串口回显、按键中断三项通关线
- 下次验收只看两件事：功能是否稳定、是否能解释基本逻辑
