# 第三阶段任务（项目能力阶段）

## 1. 阶段定位

本阶段目标是把前两阶段能力投入真实机器人子系统，形成“通信 -> 控制 -> 执行 -> 反馈”闭环开发能力。

通过后至少应具备：

- 能完成 CAN 通信收发与调试
- 能驱动电机并建立闭环控制
- 能进行 PID 调参与性能评估
- 能参与多模块联调并定位系统问题

推荐节奏：4 到 8 周，具体按项目实际需求调整。

## 2. 阶段统一规则

- 进入条件：第二阶段通过
- 验收方式：面对面验收，按总则评分
- 本阶段更强调项目接入、稳定性和排障流程
- 提交物以代码、参数表、笔记、AI 使用记录和现场验收为主

---

## 节点 1：CAN 通信与电机驱动入门

### 本节点目标

1. 完成 CAN 总线基础配置与收发验证
2. 跑通至少 1 路电机驱动链路
3. 建立通信诊断习惯（ID、帧率、超时）

### 通关线

1. 完成 CAN 初始化与收发测试
2. 实现电机基础控制（启停 / 目标值下发）
3. 输出通信映射表（ID、数据定义、来源去向）

### 提升线

- 双电机独立控制
- 基本通信异常检测（超时 / 丢帧计数）

### 预研线

- 多节点通信联调
- 极简心跳与离线检测机制

### 验收问题（抽 2 到 3 个）

- CAN 帧 ID 在你系统里的语义是什么？
- 出现电机不动时先查哪三步？
- 为什么通信层和控制层要分离？

---

## 节点 2：反馈数据与闭环控制

### 本节点目标

1. 完成反馈数据采集与滤波处理
2. 建立速度或位置闭环（至少一种）
3. 明确采样周期、控制周期与执行周期关系

### 通关线

1. 获取并解析电机反馈数据
2. 实现单环控制（速度环或位置环）
3. 输出控制周期说明与时序图

### 提升线

- 简单滤波（滑动平均 / 一阶滤波）
- 输出限幅与斜率限制

### 预研线

- 同时支持速度模式与位置模式
- 基础抗饱和策略

### 验收问题（抽 2 到 3 个）

- 控制周期为什么这样设置？
- 反馈噪声如何影响闭环表现？
- 限幅和滤波分别解决什么问题？

---

## 节点 3：PID 调参与协同控制

### 本节点目标

1. 建立规范调参流程
2. 完成双电机或多电机协同控制
3. 能识别并处理典型动态问题

### 通关线

1. 完成 PID 参数整定记录（至少 3 轮）
2. 输出性能指标（超调、稳态误差、响应时间）
3. 完成双电机或多电机协同动作演示

### 提升线

- 分段参数或工况切换参数
- 前馈补偿尝试

### 预研线

- 串级控制（位置外环 + 速度内环）
- 故障注入测试（反馈异常 / 通信抖动）

### 验收问题（抽 2 到 3 个）

- 你的调参顺序和依据是什么？
- 为什么当前参数组合可接受？
- 多电机协同时最难的问题是什么？

---

## 节点 4：系统联调与项目接入

### 本节点目标

1. 接入实际机器人子系统进行联调
2. 建立现场问题排查流程
3. 形成可交接的模块文档

### 通关线

1. 打通至少一条完整链路：通信 -> 控制 -> 执行 -> 反馈
2. 完成联调问题清单与修复记录
3. 输出模块接入文档（接口、参数、注意事项）

### 提升线

- 安全策略（失联降级、输出钳位）
- 在线日志或参数查看工具化

### 预研线

- 比赛节奏下的快速重配置与复测
- 可复用控制模块沉淀

### 验收问题（抽 2 到 3 个）

- 当前链路瓶颈在哪里？
- 现场出现异常时你的排查顺序是什么？
- 你的模块如何被其他同学安全复用？

## 3. 阶段通关标准

满足以下全部条件：

1. CAN + 电机驱动 + PID 控制链路完整
2. 能独立完成一次模块级联调闭环
3. 能沉淀参数表、故障记录与接入文档
4. 具备参与真实机器人项目开发能力

## 4. 未过线补救路径

- 先补链路完整性，再补性能优化
- 先补单环稳定，再补协同控制
- 下次验收优先看：问题定位顺序、参数解释、临时小改能力
