对于传统的使用每发送一 bit 数据就执行一次中断，（CPU 参与每次字节收发）。接收靠 `RXNE` 中断（每收到 1 字节打断 1 次 CPU），发送靠死等标志位（占用 CPU 时间），CSP 的方式是： CPU 仅做管理，接收由硬件 DMA 自动搬运到内存，只有收到一整帧（总线空闲）或缓冲区过半/满时才通知 CPU；发送先把数据写入软件 Buffer，然后一并交给 DMA 在后台发送。

初始化逻辑上，直接抛弃弱函数 `MspInit`；在基础的 HAL 库开发中，我们通常调用 `HAL_UART_Init()`，然后由 HAL 库去回调 `HAL_UART_MspInit()` 来配置引脚和时钟。但在 CSP 代码中，这种层级被打破了
直接使用`usart1_init` 函数，将所有硬件资源（GPIO、时钟、NVIC、DMA、内存分配）的初始化全部集中在 `usart1_init` 函数内部完成，并引入了 `CSP_MALLOC`。在开启 DMA 接收时，直接在堆区（Heap）动态分配 DMA 接收缓冲区（`recv_buf`）和环形缓冲区（`rx_fifo`）
```c
usart1_rx_fifo.recv_buf = CSP_MALLOC(usart1_rx_fifo.buf_size);
usart1_rx_fifo.rx_fifo = ring_fifo_init(usart1_rx_fifo.rx_fifo_buf, usart1_rx_fifo.fifo_size, RF_TYPE_STREAM);
```

初始化 DMA 句柄后，使用宏 `__HAL_LINKDMA(&usart1_handle, hdmarx, usart1_dmarx_handle);` 将 UART 句柄与 DMA 句柄物理绑定


接收上，采用DMA + 空闲中断 + 环形缓冲区 (ring_fifo)，原代码使用状态机检测 `0x0D 0x0A` 来判断一帧结束，但这种协议有局限性（不能传输包含 `0x0D` 的二进制数据）。CSP 采用的是 **IDLE（空闲）中断** 机制。

**1. 开启 DMA 循环接收**

```c
// 设置 DMA 为循环模式 (在 usart1_dmarx_handle 初始化中)
.Mode = DMA_CIRCULAR,
// 开启 DMA 接收
HAL_UART_Receive_DMA(&usart1_handle, usart1_rx_fifo.recv_buf, usart1_rx_fifo.buf_size);
```

DMA 会在后台静默地将串口收到的数据搬运到 `recv_buf`，装满后自动从头开始覆盖，全程无需 CPU 干预。

**2. 引入 IDLE 空闲中断**

```c
__HAL_UART_ENABLE_IT(&usart1_handle, UART_IT_IDLE); // 开启空闲中断
```

- **原理：** 当串口在接收完一个字节后，如果总线上持续一个字节的时间没有新的数据到来（保持高电平），硬件会判定“这一帧数据发完了”，从而触发 IDLE 中断。
    
- **触发位置：** 在 `USART1_IRQHandler` 中，代码检测到了 IDLE 标志，随后调用 `uart_dmarx_idle_callback`。
    

**3. 环形缓冲区（Ring FIFO）管理** 在 `uart_dmarx_idle_callback` 中，代码需要计算出这次 DMA 到底搬运了多少个字节，并把它们转移到更安全的环形缓冲区中供应用层读取。

C

```
/* 1. 计算当前 DMA 传输剩余量，反推出已经接收了多少字节 (tail_ptr) */
tail_ptr = huart->RxXferSize - __HAL_DMA_GET_COUNTER(huart->hdmarx);

/* 2. 计算本次新增数据的偏移和长度 */
offset = (uart_rx_fifo->head_ptr) % (uint32_t)(huart->RxXferSize);
copy = tail_ptr - offset;
uart_rx_fifo->head_ptr += copy;

/* 3. 写入环形缓冲区 */
ring_fifo_write(uart_rx_fifo->rx_fifo, huart->pRxBuffPtr + offset, copy);
```

应用层只需要随时调用 `uart_dmarx_read()` 从 FIFO 中取数据即可，完全不用管底层是怎么收的。

传统 `printf` 是阻塞的，如果发 1000 个字符，CPU 就必须在原地死等发送完毕。CSP 对此进行了彻底改造，维护了一个软件发送缓冲区 `usart1_tx_buf`
- **装填数据 (`uart_dmatx_write`)：** 当你想发送数据时，代码只是利用 `memcpy` 把数据复制到软件缓存区 `send_buf` 中。这个过程极快。
    
- **启动发送 (`uart_dmatx_send`)：** 调用此函数时，代码将缓存区的数据丢给 DMA：
```c
HAL_UART_Transmit_DMA(huart, send_tx_buf->send_buf, (uint16_t)len);
```

- 随后 CPU 立刻返回继续执行后续代码，DMA 硬件会在后台默默把数据从缓存区推向串口寄存器（`DR`）。


 `uart_printf` 也进行了重构： 不再重写标准库的 `fputc`，而是自己实现了一个 `uart_printf`。

```c
int uart_printf(UART_HandleTypeDef *huart, const char *__format, ...) {
    // ... 解析变参，格式化成字符串存入局部静态数组 uart_buffer (vsnprintf) ...
    // ... 直接调用 HAL_UART_Transmit_DMA 发送出去 ...
}
```

在基础 HAL 开发中，中断处理通常放在 `stm32f1xx_it.c` 中。但在 CSP 架构中，为了保证模块的高内聚性，中断服务函数（ISR）直接写在了驱动文件里。

```c
void USART1_IRQHandler(void) {
    if (__HAL_UART_GET_FLAG(&usart1_handle, UART_FLAG_IDLE)) { // 检测空闲中断
        __HAL_UART_CLEAR_IDLEFLAG(&usart1_handle);              // 清除标志位
        uart_dmarx_idle_callback(&usart1_handle);               // 处理数据
    }
    HAL_UART_IRQHandler(&usart1_handle); // 兜底处理其它中断(如出错)
}

void USART1_RX_DMA_IRQHandler(void) {
    HAL_DMA_IRQHandler(&usart1_dmarx_handle); // 处理 DMA 传输过半和完成中断
}
```

### 总结

1. **数据流向：** 物理引脚 → 硬件移位寄存器 → 硬件 DR 寄存器 → **DMA** → **DMA接收缓存(`recv_buf`)** → **空闲中断触发** → **环形缓冲区(`rx_fifo`)** → 应用层读取。
    
2. **效率极高：** CPU 完全不再参与单字节的数据搬运。
    
3. **协议无关：** 通过总线电平的空闲状态来切分数据帧，不再依赖特定的结束符（如回车换行），支持任意二进制数据流。

## 时序关系（RX核心）

```
时间轴 ──────────────────────────────────────────────────────────►

UART总线:  [H][e][l][l][o][\r][\n]   空闲...
                                      ↑
                                   IDLE中断触发

DMA:       ████████████████████████   持续循环接收
                                      ↑
                               halfdone/done 回调（按需）

ISR:                                  → ring_fifo_write()

task:      阻塞──────────────────────►│←唤醒→uart_dmarx_read()→处理
           vTaskDelay(10ms)                 ring_fifo_read()
```

**核心设计思想**：ISR（中断服务例程）只负责**搬数据**，任务只负责**处理数据**，ring_fifo 在中间做**解耦缓冲**，两者完全异步，互不阻塞。

## 第一步：物理比特传输（最底层）

UART 是**异步串行**协议，一个字节 `'H'`（0x48 = 0100 1000）在物理引脚上的样子：

```
PA10引脚电平:

  空闲(高)   起始位  D0  D1  D2  D3  D4  D5  D6  D7  停止位  空闲(高)
─────────┐   ┌───┬───┬───┬───┬───┬───┬───┬───┐   ┌──────────
         │   │   │   │   │   │   │   │   │   │   │
         └───┘   └───┘   └───┘   └───┘   └───┘   │
          低  0   0   0   1   0   0   1   0   高

         ← 每个bit = 1/115200 ≈ 8.68μs →
         ← 一帧总共 10 bits = 86.8μs →
```

USART1硬件模块**全自动**完成：

- 检测起始位下降沿 → 开始采样
- 按波特率时钟采样 8 个数据位
- 把并行字节放入 **DR（数据寄存器）**
- 置位 **RXNE 标志**（RX Not Empty）

## 第二步：DMA 接管 — 硬件搬运，CPU 不参与

### DMA 的硬件配置（来自代码）

```
DMA1，Channel 5  ←─── STM32F1xx 硬件固定映射：USART1_RX 只能用这个通道
```

### DMA 工作原理

```
USART1->DR (外设寄存器)          recv_buf[256] (内存)
      │                                │
      │  每当 RXNE=1（有新字节），      │
      │  DMA 自动触发一次搬运：         │
      └──────── 1 byte ───────────────►│[ptr++]
                                       │
                              ptr 到255后自动回到0
                              （循环模式，永远不停）
```

**从发送"Hello\r\n"到 recv_buf 填满的过程：**

```
时刻 t0: recv_buf = [ ][ ][ ][ ][ ][ ][ ]...  head_ptr=0
         DMA_CNDTR = 256（剩余传输数）

时刻 t1: 'H' 到达 USART1->DR，RXNE=1
         DMA 硬件自动：recv_buf[0]='H'，CNDTR=255

时刻 t2: 'e' 到达
         DMA 自动：recv_buf[1]='e'，CNDTR=254

... 依此类推 ...

时刻 t7: '\n' 到达
         DMA 自动：recv_buf[6]='\n'，CNDTR=249
         UART 总线进入空闲状态 → 触发 IDLE 中断 ↓
```

## 第三步：三种中断触发时机与函数调用链

### 触发1：IDLE 空闲线中断（最常用）

**触发条件**：一帧数据发送完毕后，RX 线持续空闲超过 **1 个完整帧时间（10 bits ≈ 86.8μs）**

```
时序：
PA10: [H][e][l][l][o][\r][\n] ──空闲86.8μs──►
                                              ↑
                               USART1->SR的IDLE位置1
                               → 触发 USART1_IRQHandler
```

[UART_STM32F1xx.c:252-259](vscode-webview://0n524cq7dpr3rhcsv07365ng6k644vr46lrh7okno3qku9htg8sa/Drivers/CSP/UART_STM32F1xx.c#L252-L259)

```c

void uart_dmarx_idle_callback(UART_HandleTypeDef *huart) {
    // tail_ptr = DMA已写入的实际位置
    // RxXferSize=256, CNDTR=249 → tail_ptr = 256-249 = 7
    tail_ptr = huart->RxXferSize - __HAL_DMA_GET_COUNTER(huart->hdmarx);

    // offset = 上次读到哪了（取模得实际下标）
    offset = uart_rx_fifo->head_ptr % huart->RxXferSize;  // = 0

    copy = tail_ptr - offset;  // = 7（"Hello\r\n"共7字节）
    uart_rx_fifo->head_ptr += copy;  // head_ptr = 7

    // 把 recv_buf[0..6] 搬入 ring_fifo
    ring_fifo_write(uart_rx_fifo->rx_fifo, huart->pRxBuffPtr + offset, copy);
}
```

## 全景硬件时序图

```
UART总线 PA10:
  ─────┐ [H]  [e]  [l]  [l]  [o]  [\r] [\n]    86μs空闲  ─────
       └────────────────────────────────────────┘

USART1->DR:  每收到1字节，RXNE=1，通知DMA

DMA CNDTR:   256→255→254→253→252→251→250→249    (停在249)

recv_buf:    [H][e][l][l][o][\r][\n][ ]...[ ]
              0  1  2  3  4   5   6  7      255

中断触发:
  空闲86μs后 → USART1_IRQHandler → uart_dmarx_idle_callback
                                     tail=7, offset=0, copy=7
                                     ring_fifo_write(buf, 0, 7)

ring_fifo:   [H][e][l][l][o][\r][\n]
              head=0              tail=7

10ms后任务醒来:
  uart_dmarx_read() → ring_fifo_read(buf, 63)
  返回: rx_buf="Hello\r\n", rx_len=7
```

**结论**：CPU 在整个接收过程中只在 **IDLE 中断** 那一瞬间（约几十个时钟周期）介入，其余时间全部是 DMA 硬件在工作，这就是 DMA+IDLE 方案相比纯中断接收效率高得多的原因。
